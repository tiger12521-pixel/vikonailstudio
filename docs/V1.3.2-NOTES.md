# V1.3.2 Gallery Metadata Fix

## 問題原因

V1.3.1 將所有 Gallery Metadata 儲存在同一個 R2 `gallery/metadata/gallery.json`。
新增作品時會先讀取整份 JSON、加入一筆，再覆寫整份 JSON。如果兩次請求時間重疊，兩次都可能讀到同一個舊版本，最後完成的寫入會蓋掉另一筆，造成已上傳作品消失。

## 修正方式

- 每件作品改存於 `gallery/items/<id>.json`。
- 新增、修改與刪除只處理該作品自己的 Metadata，不再覆寫整個作品清單。
- 舊版 `gallery/metadata/gallery.json` 仍會讀取並與新格式合併。
- 刪除舊格式作品時會建立 tombstone，避免舊資料重新出現。

## 部署

不需新增 Cloudflare Binding，也不需手動移轉 R2 資料。重新部署 V1.3.2 後即可使用。
