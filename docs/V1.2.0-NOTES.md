# Vico Nail Story v1.2.0

## 版本目標

v1.2.0 新增第一版活動內容管理系統，讓管理者不必修改 HTML、JSON 或 Git，即可上傳海報及維護最新活動。

## 新增功能

- `/admin/` 活動管理頁面
- 新增、編輯、顯示、隱藏與刪除活動
- 桌機及手機海報上傳
- 常駐活動、日期活動、置頂及排序
- 前台優先讀取 `/api/promotions`
- API 失敗時保留 `data/promotions.json` fallback
- 本機 Express 管理 API，方便正式部署前完整測試
- Cloudflare Pages Functions 管理 API
- Cloudflare D1 活動資料表
- Cloudflare R2 海報儲存
- Cloudflare Access Email One-time PIN 部署規劃

## 保留修正

- 區塊名稱使用「最新活動」
- 中文區塊標籤使用專屬字級
- 海報使用 `object-fit: contain`，資訊內容不裁切
- 既有 Booking、Gallery、Price、Booking Guide 與 FAQ 不變

## 本機測試

```bash
npm ci
npm start
```

開啟：

```text
http://localhost:3200/admin/
```

本機模式不經 Cloudflare Access，可直接測試活動 CRUD 與圖片上傳。資料會寫入 `data/promotions.json`，圖片會寫入 `assets/uploads/promotions/`。

## 正式環境

正式部署前必須完成：

1. 建立 D1 database 並執行 migrations。
2. 建立 R2 bucket。
3. 設定 Pages bindings：`DB`、`PROMOTION_IMAGES`。
4. 使用 Cloudflare Access 保護 `/admin/*` 與 `/api/admin/*`。
5. Access policy 僅允許指定的兩個 Email，登入方式使用 One-time PIN。

詳細步驟請見 `docs/CLOUDFLARE-CMS-SETUP.md`。
