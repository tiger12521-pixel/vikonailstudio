# V1.1.1 開發說明

## 本版目標

將 V1.1.0 的「最新優惠」預留區升級為資料驅動的正式活動區，並保留未來串接管理後台的擴充空間。

## 前台功能

- 桌機版載入橫式活動海報。
- 手機版載入直式活動海報。
- 點擊海報可開啟原生 Dialog 放大檢視。
- 活動提供標題、簡短說明與 LINE 預約按鈕。
- 圖片使用 Lazy Loading 與非同步解碼。
- 圖片載入失敗時顯示錯誤提示，不影響其他區塊。
- 無有效活動或資料讀取失敗時，顯示 Instagram 與 LINE 備援內容。

## 資料管理

活動資料位於：

```text
data/promotions.json
```

活動海報位於：

```text
assets/images/promotions/
├── gift-campaign-desktop.jpg
└── gift-campaign-mobile.jpg
```

每筆活動支援：

- `id`：活動唯一識別名稱。
- `title`：前台活動標題。
- `description`：活動簡短說明。
- `desktopImage`：桌機橫式海報。
- `mobileImage`：手機直式海報。
- `imageAlt`：海報完整文字描述。
- `startDate`：開始日期，格式為 `YYYY-MM-DD`。
- `endDate`：結束日期；長期活動可設為 `null`。
- `buttonText`：預約按鈕文字。
- `buttonUrl`：預約連結。
- `isActive`：是否啟用。
- `sortOrder`：多筆有效活動時的優先順序。

目前前台顯示排序後的第一筆有效活動。這次不加入輪播，避免首次資料化同時擴大互動複雜度。

## 日後更換活動

在管理後台完成前，可依下列方式手動更新：

1. 將新海報放入 `assets/images/promotions/`。
2. 修改 `data/promotions.json` 的圖片路徑與活動文字。
3. 使用 `isActive` 停用舊活動，或調整 `startDate`、`endDate`。
4. 執行 `npm start` 進行本機確認。
